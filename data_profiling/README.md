# Profile Before You Trust It

Automated data profiling as an ingestion gate: a new source file gets a full
interactive profile — missingness, duplicates, format drift, outliers —
**before** anyone writes warehouse DDL for it. Data quality rules then come
from evidence, not guesses.

## What the Notebook Does

1. **First contact** — shape, dtypes, and the smells they reveal (a `zip`
   column that lost its leading zeros, dates parsed as plain objects).
2. **Targeted checks** — per-column missing rates; exact and case-variant
   duplicate merchants.
3. **The one-liner profile** — `ProfileReport` from
   [`fg-data-profiling`](https://github.com/Data-Centric-AI-Community/fg-data-profiling)
   (the maintained successor of `pandas-profiling` / `ydata-profiling`)
   generates a self-contained interactive HTML report and a machine-readable
   alert list.
4. **Findings → rules** — each alert is mapped to a concrete ingestion
   control: completeness thresholds, dedup keys, type casts, accepted ranges.

**Explore the generated report live:**
[dymytryo.github.io — interactive profiling report](https://dymytryo.github.io/assets/reports/data-profiling-report.html)

## Files

- `profile_report.html` — generated by the notebook (not committed; see the
  hosted copy above).
