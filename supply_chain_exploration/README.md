# Three Tables, No Data Dictionary

A time-boxed exploration of a freight forwarder's raw extracts — shipments
(`files`), `containers`, and `customers` — delivered with zero context or
documentation. The notebook builds a local DuckDB sandbox, profiles every
column, untangles the keys, assembles one big table, and walks out with
revenue drivers, margin structure, and retention health.

## What the Notebook Does

1. **Local warehouse sandbox** — loads the CSVs into in-memory DuckDB, so
   exploration burns zero Snowflake credits and stays fully reproducible.
2. **Profiling, two ways** — a helper that converts Snowflake `CREATE TABLE`
   DDL into per-column SQL profiling queries (counts, distincts, null rates,
   min/max), cross-checked against one-line `ProfileReport`s written to
   `artifacts/`.
3. **Key forensics** — primary/foreign key validation across all three
   tables, orphan detection in both directions, and duplicate container
   records resolved by a most-frequent-feature-per-column strategy.
4. **OBT assembly** — `files ⟕ customers ⟕ containers_reduced` into the one
   big table downstream analysis runs on.
5. **Revenue-driver decomposition** — [`wise-pizza`](https://github.com/transferwise/wise-pizza)
   with a lasso solver explains which segments (product, carrier, lane,
   vertical, load terms) attracted or detracted net revenue between 2022
   and 2023.
6. **Margin structure** — transactions binned into size deciles show smaller
   orders carrying the strongest profit margins over time.
7. **Retention flow** — monthly new / retained / churned / resurrected
   customer units as a customer-base health check.
8. **KPI readout** — revenue, network breadth (distinct origins and
   destinations), and customer/carrier counts with fitted trendlines, ready
   to transfer to a BI tool.

## Files

- [`supply_chain_exploration.ipynb`](supply_chain_exploration.ipynb) — the
  analysis, executed end-to-end against the bundled sample.
- [`data/FILES.csv`](data/FILES.csv), [`data/CONTAINERS.csv`](data/CONTAINERS.csv),
  [`data/CUSTOMERS.csv`](data/CUSTOMERS.csv) — a deterministic **~5% customer
  sample** of the original private extracts (22,257 of 578,215 shipment
  records). Every sampled customer keeps their full order history, and the
  original referential quirks — customers present in `files` but missing from
  `customers`, and vice versa — are preserved so the notebook's integrity
  checks stay meaningful. Identifiers were already anonymized at the source.
- [`data/make_sample.py`](data/make_sample.py) — the sampling script, for
  full transparency on how the sample was drawn.
- `artifacts/` — profiling reports generated by the notebook (not committed).

## Run It

```sh
pip install duckdb pandas numpy matplotlib seaborn scikit-learn wise-pizza ydata-profiling kaleido jupyter
jupyter notebook supply_chain_exploration.ipynb
```

Runs end-to-end against the bundled sample; hash-based sampling keeps the
published outputs reproducible.
